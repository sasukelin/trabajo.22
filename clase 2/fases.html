Fase Activdad
Iniciacion
  • Actividad 1 : Nace la idea del producto de software
  • Actividad 2 : Se identifican los clientes y segmentos del mercado
  • Actividad 3 : Se entrevistan algunos clientes para identificar sus necesidades y conocer de forma general las funcionalidades que debería tener el SW.
  • Actividad 4 : Se hace un análisis para determinar si existen otros productos similares
  • Actividad 5 : Se analiza la factibilidad del producto y la posible aceptación por parte de los usuarios
Planeacion
  • Actividad 1 : Se desarrolla un plan para determinar los recursos necesarios para implementar el proyecto de software (tiempos, cronograma, personal, etc)
  • Actividad 2 : Se determina el costo aproximado del proyecto y el tiempo de desarrollo
  • Actividad 3 : Dependiendo de la fecha de lanzamiento del producto se puede ajustar el cronograma
  • Actividad 4 : El plan de desarrollo del producto se actualiza de acuerdo con las necesidades del proyecto (ej: si se incluyen nuevas funcionalidades)
  • Actividad 5 : Se desarrolla el Plan de Gestión de la Configuración: Contiene actividades para controlar los cambios y manejar versiones de los diseño
Analisis
  • Actividad 1 : Se recopila la mayor cantidad de información posible sobre las necesidades de los clientes
  • Actividad 2 : Se recopila información detallada sobre las funcionalidades del producto, desempeño requerido para el producto, confiabilidad, seguridad, usabilidad
  • Actividad 3 : Los requerimientos generales se pueden recopilar por medio de entrevistas y lluvias de ideas con los clientes
  • Actividad 4 : Los requerimientos definen qué es lo que el producto de software debe hacer
Diseño
  • Actividad 1 : Esta fase define cómo el software debe ser construido para cumplir con los requerimientos definidos en la fase anterior
  • Actividad 2 : Se define la arquitectura del software (división en sub-sistemas) y se diseña de forma específica cada componente del software. Todo esto se acompaña de la respectiva documentación que es lo que conforma el documento de diseño del software
Implementacion
  • Actividad 1 : En esta fase se programa todo el software
  • Actividad 2 :El objetivo es tomar el diseño obtenido en la fase anterior y programarlo completamente usando las tecnologías que se hayan definido
  • Actividad 3 : Como parte de la programación del software, se integran otros componentes de software como componentes de otros fabricantes
Pruebas
  • Actividad 1 : El software se prueba para determinar si cumple con los requerimientos
  • Actividad 2 :El software se prueba en tres fases:
    • 1 : Cada módulo se prueba de forma independiente
    • 2 : Los módulos se van integrando para determinar si la comunicación entre los módulos es correcta
    • 3 : Una vez se integran todos los módulos y se conforma el sistema completo, se determina si cumple con los requerimientos
